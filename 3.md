# 3-Month Roadmap — Big Tech Prep (Backend + AI Engineering + DSA)

**Target:** Big tech companies. Depth-first learning + daily DSA + one evolving project.

**Daily structure:**
- 3-4 hrs → Project building
- 1 hr → DSA (non-negotiable, every day)
- 1 hr → System design (practice out loud, interview-style)
- 1 hr → That week's focused study topic

---

## The Project (built once, evolves for 3 months)

```
Next.js (frontend)
    ↓
FastAPI (backend)
    ↓
PostgreSQL (data) + Redis (cache/sessions)
    ↓
LangGraph + One Vector DB (RAG)
    ↓
Docker → Nginx → Cloud (AWS) → CI/CD
```

---

## MONTH 1 — Backend & Infrastructure

### Week 1-2: PostgreSQL, Redis, API Design
- **Postgres:** schema design, normalization, joins, indexes (B-tree), transactions & ACID, `EXPLAIN ANALYZE`, query optimization, N+1 problem, connection pooling
- **Redis:** cache-aside pattern, TTL, session store, rate limiting
- **API/Auth:** JWT, refresh tokens, password hashing (bcrypt), REST API design, middleware
- **Build:** Full authentication system (Postgres + Redis + JWT)

### Week 3: Docker
- Multi-stage builds, image layers, image optimization, Docker volumes, Docker networks, docker-compose, health checks, secrets management, production vs. dev Dockerfiles

### Week 4: Nginx + Deployment
- Reverse proxy, load balancing, SSL/TLS, HTTPS, gzip compression, static file serving, virtual hosts
- Deploy dockerized stack behind Nginx on a VPS

---

## MONTH 2 — AI Engineering

### Week 5-6: LangGraph (agent orchestration)
- Nodes, edges, state graphs, conditional routing, memory (short-term/long-term), tool calling, human-in-the-loop, multi-agent systems
- **Build:** one real agent inside the project (e.g., support agent that looks things up and takes actions)

### Week 7: RAG (Retrieval-Augmented Generation)
- Chunking strategies, embeddings, cosine similarity, vector databases (pick **one**: ChromaDB or Qdrant), semantic search, retrieval, re-ranking, context window management
- **Build:** RAG feature into the project (e.g., "chat with your docs")

### Week 8: Tool Calling & MCP
- Function/tool calling, MCP (Model Context Protocol) — concept and basic implementation
- Agentic patterns: planning, reflection, reasoning loops

---

## MONTH 3 — Production Engineering + System Design + Cloud

### Week 9: System Design Fundamentals
- Caching strategies, CDN, database replication, database sharding, load balancing, message queues, consistent hashing, CAP theorem, horizontal vs. vertical scaling
- **Practice designs:** URL Shortener, Instagram, WhatsApp, Netflix, Uber

### Week 10: Cloud (AWS)
- EC2, RDS, S3, IAM (roles/policies), Route53, VPC basics, security groups
- Migrate project from VPS to AWS

### Week 11: Kubernetes + Observability + CI/CD
- **Kubernetes:** pods, services, deployments, ConfigMaps, secrets, ingress, persistent volumes, horizontal pod autoscaler, rolling updates
- **Monitoring:** Prometheus, Grafana
- **Logging:** Loki (or ELK)
- **CI/CD:** GitHub Actions, automated deploy pipelines

### Week 12: Polish + Documentation
- Finalize features, write architecture README explaining every design decision and tradeoff

## DSA — Daily, Throughout All 3 Months

**Pace:** 1 hour/day, ~150-200 problems total (quality over quantity). Use Neetcode roadmap / Blind 75 / Grind 169 as your problem set.

### Core Data Structures
- Arrays, Strings, Hash Maps/Sets, Linked Lists (singly/doubly), Stacks, Queues, Heaps/Priority Queues, Trees (Binary Trees, BSTs), Tries, Graphs, Union-Find (Disjoint Set)

### Core Patterns/Techniques
- Two Pointers
- Sliding Window
- Binary Search
- Fast & Slow Pointers
- Merge Intervals
- Backtracking
- Recursion
- Depth-First Search (DFS)
- Breadth-First Search (BFS)
- Topological Sort
- Dynamic Programming (1D, 2D, Knapsack patterns, DP on trees/graphs)
- Greedy Algorithms
- Bit Manipulation
- Segment Trees (advanced, optional)

### Complexity & Theory
- Big-O / time & space complexity analysis
- Recurrence relations

### Suggested Order (roughly 2-3 weeks per group)
1. Arrays/Strings → Two Pointers → Sliding Window
2. Hash Maps → Linked Lists → Stacks/Queues
3. Trees → BFS/DFS → Tries
4. Graphs → Union-Find → Topological Sort
5. Dynamic Programming → Greedy → Backtracking
6. Review + Bit Manipulation + mixed mock interviews

## System Design — Daily Practice (Interview-Style)

Practice out loud, narrate tradeoffs, 45 min per design.

**Core concepts:** Load balancers, caching (CDN, Redis), database replication & sharding, message queues (Kafka/SQS), consistent hashing, rate limiting, CAP theorem, microservices vs. monolith, API gateway, eventual consistency, idempotency

**Designs to practice:** URL Shortener, Instagram, WhatsApp/Messenger, Netflix, YouTube, Uber, Discord/Slack, Google Drive/Dropbox, ChatGPT-like system, TikTok feed

## End-of-3-Months Checklist

By the end you should be able to explain, without notes:
- [ ] Why Redis caching pattern you chose, and its tradeoffs vs. alternatives
- [ ] How your JWT + refresh token auth flow works end-to-end
- [ ] Why you picked LangGraph over raw function calling
- [ ] How your RAG pipeline chunks, embeds, retrieves, and re-ranks
- [ ] Your Docker → Nginx → AWS → CI/CD deployment pipeline, step by step
- [ ] 5+ system designs in real depth, including tradeoffs
- [ ] Core DSA patterns from memory, applied to unseen problems, not memorized solutions
